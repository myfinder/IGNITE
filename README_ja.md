# 📘 IGNITE — Intelligent Generative Networked Interaction-driven Task Engine

**IGNITE** は、複数の専門エージェントが協調して複雑なタスクを自律的に遂行する **イベント駆動型マルチエージェント実行基盤**です。
キャラクター性をもったリーダー／サブリーダーがプロジェクトを構成し、**IGNITIANS** と呼ばれる並列実行隊が並列にタスクを実行します。

## 🔥 1. 概要

IGNITE はマルチエージェントシステムであり、複数エージェントが **階層的に協調してタスクを処理**します。
リーダーが方針を定め、サブリーダーがタスク分解と管理を行い、**IGNITIANS（並列実行集団）** が実際の作業を並列で実行します。
この設計により、高度な協調・分割統治・並列性を持つ実行フローが実現できます。

## 🚀 2. 特徴

* **階層型マルチエージェント制御**
  Leader → Sub-Leaders → IGNITIANS という役割構造で振る舞いを制御します。

* **イベント駆動・自律実行**
  各ステップはイベント指向で進み、発生したイベントをトリガにして次の行動に移行します。

* **人格ベースのエージェント**
  各役割はユニットメンバーとして人格性を持ち、出力やログに反映されます。

* **並列実行と集約**
  IGNITIANS による複数タスクの同時実行が可能です。

## 🧩 3. システム構造

```
            Human User
                ↓
           Leader（伊羽 ユイ）
                ↓
      ┌───────────────┐
      ↓               ↓
Strategist     Architect     Evaluator     Coordinator     Innovator
(義賀 リオ)   (祢音 ナナ)   (衣結 ノア)   (通瀬 アイナ)  (恵那 ツムギ)
                ↓
        IGNITIANS (並列実行集団)
```

* **Leader**
  全体戦略・目標決定・最終判断。

* **Sub-Leader（5 名）**
  担当領域ごとの専門処理・実行管理。

* **IGNITIANS**
  実際の並列タスク実行部隊（個別の名前は持たない）。

## 🏷 4. メンバー紹介

### 🔥 伊羽 ユイ — Leader

**統率と鼓舞の柱。** プロジェクト全体の意思決定、目標設定・方向付け、チームの統率者。明るさと統率感を併せ持つ存在、周囲を鼓舞しながら導く、言葉で仲間を励ますタイプ

### 🧠 義賀 リオ — Strategist

**戦略家。** 大局的な戦略設計、タスク優先度付け、効率的な計画立案担当。論理的思考に長けた策士、落ち着いた判断と冷静な分析、戦略を詰めるタイプ

### 🏛 祢音 ナナ — Architect

**設計者。** 全全体設計と構造決定、モジュール/フローの最適化、他メンバーとの構造整合を担保する。美しい構造と動きの調和を重視、設計にこだわりを持つ職人肌、大局と細部の両方を把握できる

### 📊 衣結 ノア — Evaluator

**評価官。** タスク結果の検証・評価、品質チェックとテスト、基準への照合と報告を担当する。着実で几帳面、細部のチェックが得意、結果を公平に評価しつつ改善点を示す、確実さを重んじるタイプ

### 🤝 通瀬 アイナ — Coordinator

**調整役。** 全体の進行調整・リソース割り振り、サブリーダー同士の連携支援、優先順位の再調整を管理する。人と人をつなぐ調整役、柔らかいコミュニケーション力、タスク衝突の解消が得意

### 💡 恵那 ツムギ — Innovator

**創造者。** 新手法・改善策の提案、実行効率の最適化、創造的思考による革新を担当する。ひらめきと柔軟な発想力。現状に満足せず改善を探す。変化が楽しいと思えるタイプ

## ⚙️ 5. インストールとセットアップ

> 💡 Python 環境・Docker などにより動作します。
>
> 1. リポジトリをクローン
>
>    ```bash
>    git clone https://github.com/yourorg/ignite.git
>    cd ignite
>    ```
> 2. 依存パッケージのインストール
>
>    ```bash
>    pip install -r requirements.txt
>    ```
> 3. 初期設定（設定ファイル生成）
>
>    ```bash
>    ./ignite_setup.sh
>    ```

## 🛠 6. 使い方

### 🎯 起動

```bash
ignite start
```

### 🧾 プロジェクト定義

プロジェクトを開始するには、目標を指定します。

```bash
ignite plan \
  --goal "プロジェクト概要を実装する"
```

### 🚦 状態確認

```bash
ignite status
```

## 📁 7. 実装例

### ✨ 基本的なフロー

1. **ユーザー入力** を Leader が受領
2. **Leader が高レベル目標を決定**
3. **Strategist が実行計画を分解**
4. **IGNITIANS が並列にタスク実行**
5. **Evaluator が結果を評価し集約**
